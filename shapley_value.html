(train_images, train_labels), (test_images, test_labels) = keras.datasets.cifar10.load_data()

ARQUITETURA DO MODELO

Nessa etapa foi definida uma estrutura CNN para o treinamento do modelo, qualquer modelo poderia ter sido utilizado, optamos pelo uso da CNN pela maior "facilidade de implementao" e pelo treinamento ter atingido uma boa acurcia.

Modelos pr-treinados tambm foram avaliados, mas no estavam atingindo a acurcia desejada ou no tinham tanta flexibilidade em configurao, interferindo na implementao do pacote SHAP.

GERAO DOS VALORES DE SHAPLEY

Essa fase consiste na gerao dos valores de Shapley, onde cada funo pode ser explicada abaixo:

  • shap.maskers.Image = retorna uma mscara responsvel por perturbar partes da imagem, essa mscara ser usada pela funo seguinte para mensurar a importncia dos pixels na predio do modelo.
  • shap.PermutationExplainer = esse Explainer utiliza um mtodo semelhante aproximao de Monte-Carlo, a cada iterao so realizadas permutaes entre as features por outros valores do conjunto de dados de forma a aproximar os resultados do valor de Shapley real.
  • explainer = funo responsvel pelo clculo dos valores de Shapley, o parmetro outputs define a quantidade de itens que sero usados na comparao, enquanto o max_evals define a quantidade de vezes em que o modelo ser avaliado com permutaes.
In[]:
masker = shap.maskers.Image("blur(32,32)", (IMG_SIZE, IMG_SIZE, NUM_CHANNELS))
explainer = shap.PermutationExplainer(model, masker, output_names = class_names)

shap_values = explainer(test_images[IMAGE_START:IMAGE_END], max_evals = 6145, outputs = shap.Explanation.argsort.flip[:NUM_COMPARE])

INTERPRETAO E RESULTADOS

Em relao aos resultados, possvel perceber que muitos pixels de fundo esto sendo considerados nas imagens, o que pode acarretar interpretaes errneas por conta de rudos.

Outro resultado que tambm pode ser avaliado que devido a baixa resoluo das imagens (32x32), se torna mais complicado inferir alguma interpretao de pixels mais e menos relevantes para a predio.

Entretanto, a aplicao supre a ideia de demonstrar os valores de Shapley em um contexto de imagens, mostrando de forma intuititiva como os pixels podem ter sua contribuio mensurada, auxiliando em tarefas pr-treinamento e ps-treinamento.

Um detalhe importante a ressaltar que a escolha da base de dados e a resoluo aplicada se resume em dois pontos principais, um treinamento do modelo mais fcil, permitindo o foco na implementao dos valores de Shapley e reduzir o uso computacional das funes utilizadas para o clculos dos mesmos.

OBS: durante o desenvolvimento, uma tentativa foi realizada com imagens de tamanho 112x112, porm estavam causando estouro de memria com uma tentativa de alocar 42 GB, sendo necessrio ento adaptar a implementao para bases de dados com menos resolues.
